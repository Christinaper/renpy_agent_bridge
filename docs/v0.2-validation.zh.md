# v0.2 验证记录：Semantic JSON Loop Proof

## 假设

> 视觉小说运行时可以把叙事状态和玩家可用动作暴露为语义 JSON，让外部 Agent 不依赖像素级输入也能完成一轮游玩。

本验证关注桥接闭环，不验证 LLM 是否足够聪明，也不表示项目已达到生产可用。

## 结果

**日期:** 2026-05-15  
**测试游戏:** A11y-RenPy-Bridge demo  
**引擎:** Ren'Py  
**参考 Agent:** `agent/simple_player.py`  
**LLM 运行时:** Ollama  
**模型:** `llama3.2:1b`

v0.2 语义闭环成功完成。

## 有效的部分

1. `state.json` 携带了 Agent 行动所需的语义信息。
2. `turn_id` 能同步每一对 state/action。
3. `mode` 能区分游玩中和完成状态。
4. `advance` 可以表示普通对白推进。
5. `choice` 可以表示菜单决策。
6. `action.json` 能驱动 Ren'Py 推进。
7. 游戏可以不依赖视觉识别或坐标自动化完成流程。

## v0.2 可接受限制

这些限制在 Bridge Prototype 阶段可接受：

1. 阻塞等待会让 Ren'Py UI 看起来卡住。
2. 干净运行前需要清理旧的 `game/exports/*` 文件。
3. 还没有 `session_id`。
4. Ollama 超时或输出格式异常没有完整 retry/fallback。
5. choice turn 当前会复用上一句 `narrative.current_text` 作为上下文。

第 5 项是数据质量问题，但不是 v0.2 阻塞项。v0.3 应通过显式 choice context 解决。

## 验证结论

v0.2 验证了桥接原型：

```text
Semantic JSON Loop Proof Validated
```

这个结果支持项目方向：A11y 应该通过运行时拥有的语义接口实现，而不只是依赖游戏外部的视觉解释。

## 下一阶段建议验证

v0.3 应验证 Runtime Health：

- Agent 在游戏前后启动都不会读到旧状态。
- Ren'Py UI 在等待期间保持响应。
- session identity 防止跨运行污染。
- 错误会形成可观察状态，而不是静默阻塞。
- choice context 被显式表达。
