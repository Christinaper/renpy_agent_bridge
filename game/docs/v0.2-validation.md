# V0.2 Semantic JSON Loop Validation

## Hypothesis
> A visual novel game can expose narrative state and player actions as semantic JSON, enabling external agents to complete gameplay without pixel-level input.

## Validation Result ✅

**Date:** 2026-05-15  
**Test Game:** A11y-RenPy-Bridge Demo (3 scenes, 16 turns)  
**Agent:** LLM-based (llama3.2:1b via Ollama)  

### Metrics
- **Total turns:** 16
- **Advance actions:** 13 (auto-progress dialogue)
- **Choice actions:** 3 (menu selections)
- **Success rate:** 100% (completed without intervention)
- **Turn progression:** Monotonic (1→16, no gaps)
- **State machine:** Correct (awaiting_action → finished)

### Key Findings

#### ✅ What Worked
1. **JSON protocol is sufficient** - No visual input needed
2. **turn_id provides synchronization** - No race conditions observed
3. **Action typing enables automation** - Agent distinguishes advance vs choice
4. **LLM can make valid decisions** - Even simple prompts work

#### 🟡 Known Limitations (Acceptable for v0.2)
1. **Blocking wait causes UI freeze** - Game window unresponsive during turns
2. **Manual session cleanup required** - Must delete exports/ before restart
3. **No error recovery** - Any failure causes full stop

#### 🔴 Data Quality Issues (Must Fix)
1. **narrative.current_text repeats** - Last dialogue persists into choice turn

### Conclusion
**The core semantic loop is validated.** The protocol design is sound and enables full gameplay automation. Production readiness requires addressing UI responsiveness and session management (planned for v0.3).

### Reproducibility
```bash
# 1. Clear state
rm -rf game/exports/*

# 2. Start Ren'Py (Terminal 1 - Windows)
./renpy.exe game

# 3. Start Agent (Terminal 2 - WSL2)
python3 agent/simple_player.py

# Expected: Agent completes all 16 turns and exits
```
